# EigenFleet

EigenFleet 是一个基于 DevEco Studio / HarmonyOS ArkUI(ETS) 实现的对偶四元数矩阵计算与编队可行性分析应用。  
项目以论文 **Eigenvalues of Dual Hermitian Matrices with Application in Formation Control** 为理论基础，当前版本为 **beta1.6**。

本项目的目标不是只做一个“能算数值”的黑盒，而是尽量把论文中的核心算法做成一个可以直接交互、输入、验证和展示结果的移动端原型工具。

---

## 产品背景

在多智能体系统、无人机协同、移动机器人编队、空间姿态控制等场景中，研究对象往往不仅仅是简单的实矩阵或复矩阵，而是带有更强几何意义的四元数、对偶四元数及其矩阵结构。相比传统表示方式，对偶四元数能够同时统一描述旋转与平移，因此在三维刚体位姿建模、相对运动表达、队形关系分析等问题中具有天然优势。与此相对应的对偶四元数 Hermitian 矩阵特征值问题，也就不只是一个纯粹的代数问题，而是会直接影响系统结构分析、约束判定和编队可行性验证。

但这类问题在实际学习和使用中通常存在两个明显门槛。第一是理论门槛。相关内容同时涉及线性代数、四元数代数、对偶数结构和图论控制，很多使用者即使能看懂定义和公式，也不容易把它们真正落到可操作的输入、计算和验证流程上。第二是工具门槛。常见数学软件更擅长处理实数域和复数域问题，而对偶四元数矩阵通常需要研究者自行写脚本、手动组织数据格式、逐步核查中间结果，使用成本较高，也不利于教学演示和快速试验。

EigenFleet 正是在这样的背景下提出的。它希望把原本分散在论文、公式推导和实验脚本中的关键流程，尽量整合成一个可以直接交互的移动端工具。用户可以通过图形化矩阵输入组织标准部和对偶部数据，调用完整的特征值求解流程，查看结果卡片与特征向量，并进一步进行编队可行性分析，而不是停留在“读懂推导但很难上手验证”的阶段。

从产品定位上看，EigenFleet 更接近一个“研究辅助工具 + 教学演示工具 + 原型验证界面”的结合体。它不追求像大型数学平台那样覆盖所有抽象结构，而是围绕一个明确的问题域，把最核心、最常用、最容易形成实际交互价值的能力先做完整。这样既有利于降低学习门槛，也有利于后续在此基础上继续扩展更复杂的功能。

## 论文对应关系

项目主要对应论文中的两部分内容：

### 1. 特征值计算

对应论文中关于对偶 Hermitian 矩阵特征值与特征向量计算的部分，尤其是：

- 标准部 / 对偶部分裂
- 补充矩阵法（Supplement Matrix Method）
- 重根子空间中的补充矩阵构造
- 对偶部分特征值与特征向量恢复

当前实现保留了

`Quaternion -> Complex Adjoint -> Real Symmetric`

这条数值实现路线，在实数域上完成稳定的对称特征分解，再回到四元数/对偶四元数表示。

### 2. 编队可行性判定

对应论文中 dual unit gain graph 与形成控制可行性判定部分。  
当前版本实现的是 **连通图情形下的 Corollary 6.3**，会检查：

1. 零特征值是否简单且为最小特征值
2. 零特征向量各分量模长是否相等
3. `Y*LY` 是否与底层图 Laplacian `L(0)` 一致

只有三条条件同时满足时，程序才会判定目标相对构型“可行”。

## 功能概览

### A. 特征值计算模块

位置：

- `study/src/main/ets/utils/EigenSolver.ets`
- `study/src/main/ets/pages/EigenPage.ets`

支持功能：

- 输入 `2 <= n <= 8` 的对偶四元数 Hermitian 矩阵
- 标准部与对偶部独立输入
- 上三角可编辑，下三角自动按 Hermitian 共轭联动
- 随机填充测试矩阵
- 自动检查输入是否满足 Hermitian 条件
- 计算全部特征值
- 展示对应特征向量
- 使用卡片式结果展示而不是单纯文本堆叠
- 成功计算结果自动写入本地历史记录

输出形式：

- 特征值以 `lambda = lambda_s + lambda_d * ε` 的形式展示
- 数字统一保留 6 位小数并自动去除尾零
- 特征向量以多行格式展示，便于阅读

### B. 可行性判定模块

当前支持直接输入 **DQUGG 的对偶四元数 Laplacian 矩阵**。

支持功能：

- DQ Laplacian 网格输入
- 自动回填对角度数
- 平衡样例填充
- 非平衡样例填充
- 随机填充
- 基于 Corollary 6.3 的三条件判定
- 总判定 + 条件分项卡片 + 指标列表
- 中文解释气泡，帮助理解各项指标含义
- 成功判定结果自动写入本地历史记录

关键输出指标包括：

- 可行 / 不可行
- 零特征值重数
- 最小特征值
- 第二小特征值 `lambda2`
- 条件误差 `Err`
- 零特征向量
- 底层图 Laplacian `L(0)` 预览

## 当前算法实现说明

### 1. 输入解析

每个矩阵元使用 8 个实数编码：

`a,b,c,d,e,f,g,h`

其中：

- `a,b,c,d` 对应标准部四元数 `a + bi + cj + dk`
- `e,f,g,h` 对应对偶部四元数 `e + fi + gj + hk`

解析后统一进入内部的对偶四元数矩阵结构。

### 2. Hermitian 校验

在进行特征计算前，程序会检查：

- 对角元虚部是否为 0
- 非对角元是否满足共轭转置关系
- 结构是否符合 Hermitian 矩阵要求

对于可行性判定模块，还会额外校验：

- 是否为 DQUGG Laplacian
- 主对角是否为非负实数
- 非对角是否为 `0` 或 `-q_ij`
- 非零边增益是否为单位对偶四元数
- 主对角是否与边数一致

### 3. 特征值求解

标准部矩阵先被嵌入到实对称矩阵中，再通过 Jacobi 迭代完成对称特征分解。  
之后按标准特征值聚类，在重根子空间中构造补充矩阵：

`S = U^T A_d U`

再对 `S` 做谱分解，以得到对应的对偶部分特征值，并恢复对应特征向量。

### 4. 特征向量恢复

当前版本不再只是“给出近似特征值”，而是同时恢复特征向量。  
实现中使用了增广系统与残差校验来保证结果可靠性，会检查：

- `A_s x_s ≈ lambda_s x_s`
- `(A_s - lambda_s I)x_d ≈ (lambda_d I - A_d)x_s`

若残差超出阈值，会返回失败信息，而不是静默输出不可靠结果。

### 5. 可行性判定

当前实现严格围绕论文连通图版 Corollary 6.3 展开，而不是仅凭“零特征值个数”做启发式判断。  
因此，程序会分别给出：

- 条件 (i) 是否成立
- 条件 (ii) 是否成立
- 条件 (iii) 是否成立

并在界面上配合中文说明气泡解释这些指标的含义。

## 界面与交互

应用当前包含一个标题页和两个功能页：

### 1. 标题页

- 启动应用后首先进入标题页
- 顶部显示应用主标题与副标题
- 下方两个大按钮分别进入两个功能页
- 页面右下角显示当前版本号

### 2. 特征值计算

- 从标题页进入特征值计算页
- 页面左上角提供返回按钮
- 输入标准部矩阵
- 输入对偶部矩阵
- 点击“计算全部特征值”
- 结果以卡片形式呈现
- 点击按钮可查看某个特征值对应的特征向量
- 页面内可展开查看最近 10 条历史记录

### 3. 可行性判定

- 从标题页进入可行性判定页
- 页面左上角提供返回按钮
- 输入或填充 DQ Laplacian
- 可使用平衡样例、非平衡样例或随机填充
- 点击“计算并判定”
- 查看总结论、条件卡片和指标说明
- 页面内可展开查看最近 10 条历史记录

设计目标是尽量降低论文算法的使用门槛，让用户不必直接手写长串文本矩阵。

## 已知限制

- 当前判定模块主要面向连通图情形
- 输入规模目前控制在 `2 <= n <= 8`
- 随机填充主要用于快速试验，并不保证生成具有特定理论意义的 benchmark
- 历史记录当前只保存成功结果，不保存失败尝试
- 结果受浮点数误差影响，极小噪声会被自动归零处理
- 当前仍偏研究原型性质，不是工业级高性能矩阵库

## 应用前景

从短期来看，EigenFleet 已经具备成为一个轻量级实验与教学辅助工具的基础。对于正在学习对偶四元数矩阵、编队控制和图结构判定的用户，它可以把抽象的数学对象转换为更直观的输入网格、结果卡片和解释型文本，帮助用户更快理解“输入是什么”“结果意味着什么”“为什么当前判定可行或不可行”。对于研究者和开发者，它也能承担快速试算、样例复现、输入检查和结果展示的角色，减少频繁切换文档、脚本和计算环境的成本。

从中期来看，EigenFleet 有机会发展成一个面向多智能体结构分析的小型工作台。当前版本已经具备了完整的基本链路：输入矩阵、运行算法、解释结果、保存历史、回填复现。这意味着它不仅能“算一次”，还能够支持反复试验、对比不同输入、观察结构变化带来的影响。如果继续扩展样例库、图结构输入方式、结果可视化和更多数值分析入口，它就可以逐步从单点求解器成长为更系统的分析平台。

从应用落地方向来看，这类工具对多个场景都有潜在价值。在科研环境中，它适合用于论文方法复现、实验样例核查和课堂展示；在工程原型阶段，它可以作为算法前端验证工具，在尚未接入完整业务系统前提前打通交互链路；在教学场景中，它也能够帮助学生更直观地理解对偶四元数、Hermitian 结构、特征值分解和编队判定之间的关系。对于那些不希望一开始就搭建复杂数学软件环境的用户，这种移动端原型工具尤其有吸引力。

从长期演进角度看，EigenFleet 还可以继续向“结构设计与优化工具”方向推进。README 中已经提到的“编队生成器”和“编队优化系统”，本质上就是从“分析已有输入”进一步走向“辅助构造满足要求的输入”和“对不可行输入给出改进建议”。如果未来再叠加图结构编辑、编队几何可视化、参数约束设置、自动样例生成乃至更智能的优化算法，EigenFleet 将不只是一个求解器界面，而可能成长为一个面向编队分析、设计与解释的综合性工具。

当然，就当前阶段而言，EigenFleet 仍然更偏向研究原型和方法验证工具，而不是工业级的高性能数值平台。但正因为它已经具备较完整的算法主链路、较清晰的页面结构以及可继续扩展的功能边界，所以无论后续朝着教学演示、科研辅助还是工程原型支持方向发展，都已经有了不错的起点。
