# EigenFleet

## 多智能体时代的数值引擎

在高密度协作系统里，编队不只是“队形美观”的问题，而是稳定性、可控性、鲁棒性与任务效率的共同底座。  
从无人机集群、无人车协同，到海空跨域协作与分布式机器人网络，多智能体编队正在从实验室走向真实复杂场景。

EigenFleet 的核心目标，是把抽象的对偶四元数矩阵谱理论，变成一个可交互、可计算、可复现实验结果的工程工具：  
**输入一个 n 阶对偶四元数埃尔米特矩阵（2 ≤ n ≤ 8），输出其全部特征值。**

---

## 项目定位

EigenFleet 当前版本聚焦于一个核心能力：

- 对偶四元数埃尔米特矩阵特征值求解
- 采用补充矩阵法（Supplement Matrix Method）实现
- 支持 UI 可视化矩阵输入（标准部 + 对偶部）
- 输出格式为 `a+b·ε`

## 后端计算逻辑（实现说明）

后端主要实现在：

- `study/src/main/ets/utils/EigenSolver.ets`

整体流程如下。

### 1) 输入解析与结构化

每个矩阵元素按 8 个实数编码：

- `a,b,c,d,e,f,g,h`
- 前四个是标准部四元数 `a+bi+cj+dk`
- 后四个是对偶部四元数 `e+fi+gj+hk`

解析后统一进入 `DualQuaternion[][]` 数据结构。

### 2) 埃尔米特合法性校验

在进入谱计算前，先做严格校验：

- 对角元虚部必须为 0（标准部与对偶部都检查）
- 上下三角必须满足共轭转置关系  
  `Aji = conjugate(Aij)`

这一步保证输入满足理论假设，避免“看似可算、实则无意义”的结果。

### 3) 标准部 / 对偶部分离

矩阵 \( A \) 被拆成：

- 标准部 \( A_s \)
- 对偶部 \( A_d \)

后续谱计算围绕 \( A_s \) 的重根子空间展开，再由补充矩阵提取 \( A_d \) 对应信息。

### 4) 四元数矩阵嵌入到实对称矩阵

实现中先走“Quaternion → Complex Adjoint → Real Symmetric”路线：

1. 四元数埃尔米特矩阵映射到复厄米伴随矩阵；
2. 复厄米矩阵再映射到实对称矩阵。

这样可在实数域上使用稳定且成熟的对称特征分解算法。

### 5) Jacobi 特征分解

对实对称矩阵采用 Jacobi 旋转法：

- 每轮选择最大非对角元做旋转消元
- 迭代收敛后得到实特征值与特征向量
- 最终按升序排序

### 6) 补充矩阵法提取对偶特征部分

核心步骤：

1. 按标准部特征值聚类，识别重根子空间；
2. 在每个子空间内构造补充矩阵  
   \( S = U^T A_d U \)；
3. 对 \( S \) 求特征值并按重数折叠；
4. 与标准部特征值一一配对，得到  
   \( \lambda = \lambda_s + \lambda_d \varepsilon \)。

## 前端输入方式（使用方法）

页面采用可视化矩阵输入，避免手写长字符串：

- 主对角线：单框输入（实部）
- 非对角元素：2×2 小框输入（a,b,c,d）
- 上三角可编辑，下三角自动按埃尔米特共轭联动
- 标准部与对偶部分别独立输入（上下两块矩阵）
- 矩阵区域支持拖动/滚动，适配高阶输入

### 快速上手

1. 选择 `n`（2~8）；
2. 点击“填充示例”可自动随机生成标准部与对偶部；
3. 也可手工在小框里逐项修改；
4. 点击“计算全部特征值”；
5. 在结果区读取 `a+b·ε` 形式的特征值。

---

## 数值与工程约束

- 支持阶数范围：`2 <= n <= 8`
- 输入非法会给出明确错误信息（行列数、非数字、非埃尔米特结构）
- 输出数字保留到小数点后 9 位并做 `-0` 归一化处理

## 结语

EigenFleet 不是“把矩阵丢进去就结束”的黑盒。  
它强调的是：**理论结构可见、输入语义可见、数值过程可追踪**。

当多智能体系统越来越复杂，我们更需要这种兼顾数学严谨与工程可操作性的工具。  
这正是 EigenFleet 的价值所在。
