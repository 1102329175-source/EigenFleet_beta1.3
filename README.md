# EigenFleet

EigenFleet 是一个基于 DevEco Studio / HarmonyOS ArkUI(ETS) 实现的对偶四元数矩阵计算与编队可行性分析应用。  
项目以论文 **Eigenvalues of Dual Hermitian Matrices with Application in Formation Control** 为理论基础，当前版本为 **beta1.6**。

本项目的目标不是只做一个“能算数值”的黑盒，而是尽量把论文中的核心算法做成一个可以直接交互、输入、验证和展示结果的移动端原型工具。

## 版本状态

**beta1.6 已实现的核心能力：**

- 对偶四元数 Hermitian 矩阵的全部特征值计算
- 对应特征向量的数值恢复与展示
- 按论文思路实现的补充矩阵法主流程
- 基于论文 Corollary 6.3 的连通图可行性判定
- ArkUI 图形化矩阵输入、样例填充、随机填充、结果卡片展示
- 标题页 + 两个功能页的三页导航结构
- 两个功能页各自独立的本地历史记录（最近 10 条）

**当前仍未覆盖或仅部分覆盖的内容：**

- 论文中更一般的多连通分量情形（如 Theorem 6.2 的完整推广）

- 原生四元数域特征分解器，当前实现仍采用实嵌入数值路线
- 更完整的实验复现实例库与自动化回归数据集

因此，beta1.6 可以视为一个“核心算法已打通、页面动线更完整、交互体验继续完善中”的研究型应用原型。

## 论文对应关系

项目主要对应论文中的两部分内容：

### 1. 特征值计算

对应论文中关于对偶 Hermitian 矩阵特征值与特征向量计算的部分，尤其是：

- 标准部 / 对偶部分裂
- 补充矩阵法（Supplement Matrix Method）
- 重根子空间中的补充矩阵构造
- 对偶部分特征值与特征向量恢复

当前实现保留了

`Quaternion -> Complex Adjoint -> Real Symmetric`

这条数值实现路线，在实数域上完成稳定的对称特征分解，再回到四元数/对偶四元数表示。

### 2. 编队可行性判定

对应论文中 dual unit gain graph 与形成控制可行性判定部分。  
当前版本实现的是 **连通图情形下的 Corollary 6.3**，会检查：

1. 零特征值是否简单且为最小特征值
2. 零特征向量各分量模长是否相等
3. `Y*LY` 是否与底层图 Laplacian `L(0)` 一致

只有三条条件同时满足时，程序才会判定目标相对构型“可行”。

## 功能概览

### A. 特征值计算模块

位置：

- `study/src/main/ets/utils/EigenSolver.ets`
- `study/src/main/ets/pages/EigenPage.ets`

支持功能：

- 输入 `2 <= n <= 8` 的对偶四元数 Hermitian 矩阵
- 标准部与对偶部独立输入
- 上三角可编辑，下三角自动按 Hermitian 共轭联动
- 随机填充测试矩阵
- 自动检查输入是否满足 Hermitian 条件
- 计算全部特征值
- 展示对应特征向量
- 使用卡片式结果展示而不是单纯文本堆叠
- 成功计算结果自动写入本地历史记录

输出形式：

- 特征值以 `lambda = lambda_s + lambda_d * ε` 的形式展示
- 数字统一保留 6 位小数并自动去除尾零
- 特征向量以多行格式展示，便于阅读

### B. 可行性判定模块

当前支持直接输入 **DQUGG 的对偶四元数 Laplacian 矩阵**。

支持功能：

- DQ Laplacian 网格输入
- 自动回填对角度数
- 平衡样例填充
- 非平衡样例填充
- 随机填充
- 基于 Corollary 6.3 的三条件判定
- 总判定 + 条件分项卡片 + 指标列表
- 中文解释气泡，帮助理解各项指标含义
- 成功判定结果自动写入本地历史记录

关键输出指标包括：

- 可行 / 不可行
- 零特征值重数
- 最小特征值
- 第二小特征值 `lambda2`
- 条件误差 `Err`
- 零特征向量
- 底层图 Laplacian `L(0)` 预览

## 当前算法实现说明

### 1. 输入解析

每个矩阵元使用 8 个实数编码：

`a,b,c,d,e,f,g,h`

其中：

- `a,b,c,d` 对应标准部四元数 `a + bi + cj + dk`
- `e,f,g,h` 对应对偶部四元数 `e + fi + gj + hk`

解析后统一进入内部的对偶四元数矩阵结构。

### 2. Hermitian 校验

在进行特征计算前，程序会检查：

- 对角元虚部是否为 0
- 非对角元是否满足共轭转置关系
- 结构是否符合 Hermitian 矩阵要求

对于可行性判定模块，还会额外校验：

- 是否为 DQUGG Laplacian
- 主对角是否为非负实数
- 非对角是否为 `0` 或 `-q_ij`
- 非零边增益是否为单位对偶四元数
- 主对角是否与边数一致

### 3. 特征值求解

标准部矩阵先被嵌入到实对称矩阵中，再通过 Jacobi 迭代完成对称特征分解。  
之后按标准特征值聚类，在重根子空间中构造补充矩阵：

`S = U^T A_d U`

再对 `S` 做谱分解，以得到对应的对偶部分特征值，并恢复对应特征向量。

### 4. 特征向量恢复

当前版本不再只是“给出近似特征值”，而是同时恢复特征向量。  
实现中使用了增广系统与残差校验来保证结果可靠性，会检查：

- `A_s x_s ≈ lambda_s x_s`
- `(A_s - lambda_s I)x_d ≈ (lambda_d I - A_d)x_s`

若残差超出阈值，会返回失败信息，而不是静默输出不可靠结果。

### 5. 可行性判定

当前实现严格围绕论文连通图版 Corollary 6.3 展开，而不是仅凭“零特征值个数”做启发式判断。  
因此，程序会分别给出：

- 条件 (i) 是否成立
- 条件 (ii) 是否成立
- 条件 (iii) 是否成立

并在界面上配合中文说明气泡解释这些指标的含义。

## 界面与交互

应用当前包含一个标题页和两个功能页：

### 1. 标题页

- 启动应用后首先进入标题页
- 顶部显示应用主标题与副标题
- 下方两个大按钮分别进入两个功能页
- 页面右下角显示当前版本号

### 2. 特征值计算

- 从标题页进入特征值计算页
- 页面左上角提供返回按钮
- 输入标准部矩阵
- 输入对偶部矩阵
- 点击“计算全部特征值”
- 结果以卡片形式呈现
- 点击按钮可查看某个特征值对应的特征向量
- 页面内可展开查看最近 10 条历史记录

### 3. 可行性判定

- 从标题页进入可行性判定页
- 页面左上角提供返回按钮
- 输入或填充 DQ Laplacian
- 可使用平衡样例、非平衡样例或随机填充
- 点击“计算并判定”
- 查看总结论、条件卡片和指标说明
- 页面内可展开查看最近 10 条历史记录

设计目标是尽量降低论文算法的使用门槛，让用户不必直接手写长串文本矩阵。

- DevEco Studio 6.x
- HarmonyOS Stage Model
- ArkUI (ETS)

1. 使用 DevEco Studio 打开本项目根目录

- `n = 2` 的简单 Hermitian 矩阵
- 含重根的标准部矩阵

- 平衡 DQUGG 样例

## 已知限制

- 当前判定模块主要面向连通图情形
- 输入规模目前控制在 `2 <= n <= 8`
- 随机填充主要用于快速试验，并不保证生成具有特定理论意义的 benchmark
- 历史记录当前只保存成功结果，不保存失败尝试
- 结果受浮点数误差影响，极小噪声会被自动归零处理
- 当前仍偏研究原型性质，不是工业级高性能矩阵库
